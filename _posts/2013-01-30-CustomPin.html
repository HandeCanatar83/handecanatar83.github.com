--- title: IOS 6 ile Custom Pin Yapalım. layout: post description: "IOS 6 ile Custom Pin Yapalım." keywords: IOS6, MKMapView, custom pin,MKPointAnnotation,CLLocationCoordinate2D. ---
iRate sayesinde kullanıcılarımıza uygulamamızı oylatabiliriz.

IOS-Application-Single View Application'ı seçerek yeni bir uygulama oluşturuyoruz. Ve ViewControllerımıza MKMapView ekliyoruz. MKMapView'ı eklediğinizde tüm Controllerı kaplar ben Mapimi biraz küçülttüm.

Bulunduğumuz noktayı göstermesi için attributes inspector'dan Shows User Location Behavior'ı checkliyoruz.

Bu durumda uygulamamızı çalıştırdığımızda hata alacağız. MapKit framework'ünü ekleyelim.

Uygulamamızı çalıştıralım ve Konum bilgimizin alınmasını onaylıyalım. Simulatörümüz gerçekten konum bilgisi alamadığından konumumuz Apple'ın merkezi üzerinde olacaktır :)

Artık Custom Pin yapabiliriz. Map'imize pin ekleyebilmek için bir property oluşturup @synthesize edelim.

ViewController'ımızın viewDidLoad'una aşağıdaki kodları ekliyoruz.

{% highlight objc %} #import "ViewController.h" @interface ViewController () @end @implementation ViewController @synthesize mapim; - (void)viewDidLoad { [super viewDidLoad]; // Do any additional setup after loading the view, typically from a nib. CLLocationCoordinate2D annotationCoord; annotationCoord.latitude = 41.244; annotationCoord.longitude = 29.23; MKPointAnnotation *annotationPoint = [[MKPointAnnotation alloc] init]; annotationPoint.coordinate = annotationCoord; annotationPoint.title = @"Köprü"; annotationPoint.subtitle = @"Boğaziçi köprüsü"; [mapim annotationVisibleRect]; [mapim addAnnotation:annotationPoint]; [mapim setCenterCoordinate:annotationPoint.coordinate]; } {% endhighlight %}
{% highlight objc %} //AppDelegate.m #import "AppDelegate.h" #import "iRate.h"//--3 #import "ViewController.h"//--4 @implementation AppDelegate @synthesize viewController;//--5 @synthesize window;//--6 + (void)initialize { //--7 [iRate sharedInstance].applicationBundleID = @"com.bundleım.uygulamaismi"; // bundleım : provisioning portalda daha önce oluşturduğunuz Bundle Identifier (App ID Suffix) // uygulamaismi : uygulama ismi işte :) [iRate sharedInstance].onlyPromptIfLatestVersion = NO; //enable preview mode [iRate sharedInstance].previewMode = YES; //Test uygulamasında çalıştığımız için 5 gün beklemeyeceğiz ve iRate hemen çalışacak. [iRate sharedInstance].daysUntilPrompt = 5; [iRate sharedInstance].usesUntilPrompt = 15; //--7 } - (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions { // Override point for customization after application launch. //--8 [window addSubview:viewController.view]; [window makeKeyAndVisible]; //--8 return YES;; } {% endhighlight %}

Uygylamayı build ettiğinizde StoreKit framework hatası alacağız. Ve StoreKit framework'ünü ekleyelim

Ve... Bitti. IRate..

Eğer Itunes Connect Hesabınızda aktif bir uygulamanız yoksa, yani geçerli bir [iRate sharedInstance].applicationBundleID = @"com.bundleım.uygulamaismi" yoksa. IRate uygulamaya ulaşamadığından hata verecektir. Ben örnek olarak gösterebilmek için valid bir applicationBundleID kullandım.
